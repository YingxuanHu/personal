Android App with Kotlin: Campus Dating

Github Repo

I collaborated with a team to develop an Android dating application for Waterloo students using Kotlin JetPack and Git workflows.

This project is developed to help UW specific students to meet/date with people based on their preferences.

My contributions included designing and integrating multiple user interfaces to enhance the overall user experience. Additionally, we utilized Google Firebase to store user data and implemented an efficient algorithm to match users based on their preferences.

Walkthrough:

1. Login page, and Account registration

2. After Login, proceed to Profile setting, and Preferences setting

3. Now, the user can view others after matching algorithm's filtered result, while other users can see/like "Alvin" profile as well

4. If two users both liked each other, they will be able to see them in the chat section


Waterloo Special Monopoly: "Watopoly"

Github Repo

I was work on the Monopoly Coding Project with another teammate, where we created a simple modified Monopoly game using C++. The game is specific to the University of Waterloo Campus, and there are added rules as well, the specification of this project can be found here

We efficiently utilized Object-Oriented Programming (OOP) features in C++, implementing polymorphism, and also employed encapsulation by declaring critical game data as private within their respective classes.

The detailed design logic can be found here

Walkthrough:

Command Interpretation:

  • roll (used if the player can roll): the player rolls two dice, moves the sum of the two dice and takes action on the square they landed on
  • next (used if the player cannot roll): give control to the next player.
  • trade "name" "give" "receive": offers a trade to name with the current player offering give and requesting receive, where give and receive are either amounts of money or a property name. Responses are accept and reject
  • "property" buy/sell: attempts to buy or sell an improvement for property
  • mortgage "property": attempts to mortgage property.
  • unmortgage "property": attempts to unmortgage property.
  • bankrupt: player declares bankruptcy. This command is only available when a player must pay more money than they currently have
  • assets: displays the assets of the current player. Does not work if the player is deciding how to pay Tuition.
  • all: displays the assets of every player. For verifying the correctness of your transactions. Does not work if a player is deciding how to pay Tuition
